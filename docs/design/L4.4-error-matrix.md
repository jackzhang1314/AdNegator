## 1. 错误分类

### 1.1 错误层次

```
系统错误 (5xx)
  ├─ 服务不可用
  └─ 内部错误

客户端错误 (4xx)
  ├─ 认证错误 (401)
  ├─ 授权错误 (403)
  ├─ 资源不存在 (404)
  ├─ 冲突 (409)
  └─ 参数错误 (400)
```

## 2. 完整错误矩阵

| 错误代码 | HTTP | 描述 | 业务规则 | 场景 | 恢复建议 |
|----------|------|------|----------|------|----------|
| `unauthorized` | 401 | API Key 无效或过期 | BR-012 | 认证失败 | 检查 API Key 是否正确 |
| `insufficient_permissions` | 403 | 权限不足 | BR-013 | 权限范围不足 | 更新 API Key scopes |
| `quota_exceeded` | 403 | 配额超限 | BR-021, BR-081 | 达到配额上限 | 升级计划或等待配额重置 |
| `rate_limit_exceeded` | 429 | 请求过多 | BR-090 | 超过速率限制 | 降低请求频率 |
| `sandbox_not_found` | 404 | 沙盒不存在 | - | 沙盒已删除或不存在 | 检查 sandbox_id |
| `template_not_found` | 404 | 模板不存在 | - | 模板 ID 错误 | 使用 GET /templates 查询 |
| `user_not_found` | 404 | 用户不存在 | - | 用户已删除 | - |
| `invalid_parameters` | 400 | 参数错误 | BR-022, BR-030, BR-031 | 参数验证失败 | 检查请求参数 |
| `invalid_state` | 409 | 状态不允许操作 | BR-023, BR-060 | 状态转换非法 | 等待状态变更 |
| `already_paused` | 409 | 沙盒已暂停 | BR-060 | 重复暂停 | 无需操作 |
| `template_in_use` | 409 | 模板使用中 | BR-024 | 模板被引用 | 先删除使用此模板的沙盒 |
| `file_too_large` | 413 | 文件过大 | BR-050 | 文件超过 100MB | 压缩或分片上传 |
| `path_forbidden` | 403 | 路径禁止访问 | BR-051 | 访问系统路径 | 仅访问 /workspace |
| `disk_full` | 507 | 磁盘空间不足 | BR-052 | 磁盘已满 | 清理文件或增加配额 |
| `process_limit_exceeded` | 429 | 进程数超限 | BR-040 | 并发进程过多 | 等待进程结束 |
| `process_timeout` | 408 | 进程超时 | BR-041 | 执行时间过长 | 优化代码或增加超时 |
| `internal_error` | 500 | 服务器错误 | - | 未知错误 | 联系技术支持 |
| `database_error` | 500 | 数据库错误 | - | 数据库连接失败 | 稍后重试 |
| `k8s_api_error` | 500 | K8s API 错误 | - | K8s 调用失败 | 稍后重试 |
| `s3_error` | 500 | 对象存储错误 | - | S3 操作失败 | 稍后重试 |
| `criu_error` | 500 | CRIU 操作失败 | - | 检查点失败 | 重新创建沙盒 |
| `email_already_exists` | 409 | 邮箱已注册 | BR-001 | 用户注册 | 使用其他邮箱 |
| `weak_password` | 400 | 密码强度不足 | BR-002 | 用户注册 | 使用更强密码 |
| `api_key_limit_exceeded` | 403 | API Key 数量超限 | BR-011 | 创建 API Key | 删除旧的 API Key |

## 3. 错误响应格式

### 3.1 标准格式

```json
{
  "error": {
    "code": "quota_exceeded",
    "message": "Monthly sandbox quota exceeded: 105/100 used",
    "details": {
      "current_usage": 105,
      "quota_limit": 100,
      "plan": "free"
    }
  }
}
```

### 3.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `error.code` | string | ✅ | 错误代码（机器可读） |
| `error.message` | string | ✅ | 错误描述（人类可读） |
| `error.details` | object | ❌ | 额外错误详情 |

## 4. 错误处理示例

### 4.1 客户端处理

**TypeScript**:
```typescript
try {
  const sandbox = await Sandbox.create({ template: 'python-3.11' });
} catch (error) {
  if (error.code === 'quota_exceeded') {
    console.error('配额不足，请升级计划');
    // 引导用户升级
  } else if (error.code === 'rate_limit_exceeded') {
    console.error('请求过快，等待 60 秒后重试');
    await sleep(60000);
    // 重试
  } else {
    console.error('未知错误:', error.message);
  }
}
```

### 4.2 服务端处理

**FastAPI**:
```python
from fastapi import HTTPException

class BusinessRuleViolation(HTTPException):
    def __init__(self, code: str, message: str, details: dict = None):
        super().__init__(
            status_code=self.get_status_code(code),
            detail={
                "error": {
                    "code": code,
                    "message": message,
                    "details": details or {}
                }
            }
        )

    @staticmethod
    def get_status_code(code: str) -> int:
        STATUS_CODE_MAP = {
            "unauthorized": 401,
            "insufficient_permissions": 403,
            "quota_exceeded": 403,
            "sandbox_not_found": 404,
            "invalid_parameters": 400,
            "invalid_state": 409,
            # ...
        }
        return STATUS_CODE_MAP.get(code, 500)
```

## 5. 错误监控和告警

### 5.1 关键错误指标

| 指标 | 阈值 | 告警级别 |
|------|------|----------|
| 5xx 错误率 | > 1% | Critical |
| 4xx 错误率 | > 10% | Warning |
| `internal_error` 次数 | > 100/min | Critical |
| `k8s_api_error` 次数 | > 50/min | High |
| `criu_error` 次数 | > 10/min | High |

### 5.2 Prometheus 查询

```promql
# 5xx 错误率
rate(http_requests_total{status=~"5.."}[5m])
/ rate(http_requests_total[5m])

# 按错误代码分组
sum by (error_code) (rate(api_errors_total[5m]))
```

## 6. 错误恢复策略

### 6.1 自动重试矩阵

| 错误代码 | 是否重试 | 最大次数 | 退避策略 |
|----------|----------|----------|----------|
| `database_error` | ✅ | 3 | 指数退避 (2s, 4s, 8s) |
| `k8s_api_error` | ✅ | 3 | 指数退避 |
| `s3_error` | ✅ | 3 | 指数退避 |
| `rate_limit_exceeded` | ✅ | 1 | 60s 后 |
| `quota_exceeded` | ❌ | - | - |
| `invalid_parameters` | ❌ | - | - |
| `sandbox_not_found` | ❌ | - | - |

### 6.2 降级策略

| 错误场景 | 降级方案 |
|----------|----------|
| CRIU 暂停失败 | 标记为 failed，建议重新创建 |
| K8s API 超时 | 使用缓存状态，后台同步 |
| S3 不可用 | 暂时禁用暂停功能 |

## 附录

### A. E2B 兼容性

本错误代码体系**兼容** E2B API，以下为对应关系：

| E2B 错误码 | gVisor E2B 错误码 |
|-----------|-------------------|
| `SANDBOX_NOT_FOUND` | `sandbox_not_found` |
| `INVALID_API_KEY` | `unauthorized` |
| `QUOTA_EXCEEDED` | `quota_exceeded` |
| `INTERNAL_ERROR` | `internal_error` |

### B. 错误码快速查询

**按 HTTP 状态码**:
- 400: `invalid_parameters`, `weak_password`
- 401: `unauthorized`
- 403: `insufficient_permissions`, `quota_exceeded`, `path_forbidden`, `api_key_limit_exceeded`
- 404: `sandbox_not_found`, `template_not_found`, `user_not_found`
- 408: `process_timeout`
- 409: `invalid_state`, `already_paused`, `template_in_use`, `email_already_exists`
- 413: `file_too_large`
- 429: `rate_limit_exceeded`, `process_limit_exceeded`
- 500: `internal_error`, `database_error`, `k8s_api_error`, `s3_error`, `criu_error`
- 507: `disk_full`
