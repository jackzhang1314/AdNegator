## 2. SDK错误类型详细说明

### 2.1 TimeoutError (超时错误)

**TypeScript**:
```typescript
class TimeoutError extends SandboxError {
  name: 'TimeoutError'
  message: string
  code?: string
}
```

**Python**:
```python
class TimeoutException(SandboxException):
    pass
```

**触发场景**:
- Sandbox超时 (超过配置的timeoutMs)
- gRPC请求超时 (DEADLINE_EXCEEDED)
- 连接取消 (CANCELED)
- 服务不可用 (UNAVAILABLE)

**HTTP状态码**: 502 (Bad Gateway - sandbox timeout)

**gRPC状态码**:
- `DEADLINE_EXCEEDED` - 请求超时
- `CANCELED` - 请求取消
- `UNAVAILABLE` - 服务不可用

**恢复建议**:
- 增加sandbox的timeoutMs配置
- 增加命令的timeoutMs配置
- 检查网络连接
- 稍后重试

**TypeScript**:
```typescript
class InvalidArgumentError extends SandboxError {
  name: 'InvalidArgumentError'
  statusCode: 400
  message: string
}
```

**触发场景**:
- 请求参数格式错误
- 缺少必填参数
- 参数值超出范围
- 无效的模板ID或Sandbox ID

**HTTP状态码**: 400 (Bad Request)

**gRPC状态码**: `INVALID_ARGUMENT`

**恢复建议**:
- 检查API文档验证参数格式
- 确保所有必填字段都已提供
- 验证参数值在允许范围内

### 2.10 CommandExitError (命令退出错误)

**TypeScript**:
```typescript
class CommandExitError extends CommandResult {
  name: 'CommandExitError'
  exitCode: number      // !== 0
  stdout: string
  stderr: string
  error?: string
  message: string
}
```

**触发场景**:
- 命令执行返回非零退出码
- 进程异常终止
- 命令运行失败

**特殊性**:
- 继承自`CommandResult`而非`SandboxError`
- 包含完整的stdout/stderr输出
- 可以通过exitCode判断具体错误类型

**使用示例**:
```typescript
try {
  const result = await sandbox.commands.run('exit 1')
} catch (error) {
  if (error.name === 'CommandExitError') {
    console.log(`Exit code: ${error.exitCode}`)
    console.log(`Stdout: ${error.stdout}`)
    console.log(`Stderr: ${error.stderr}`)
  }
}
```

## 4. 错误响应格式

### 4.1 REST API响应格式 (JSON) - E2B Official

**E2B官方错误格式** (简单格式):
```json
{
  "code": 429,
  "message": "Monthly sandbox quota exceeded: 105/100 used"
}
```

**说明**:
- `code`: HTTP状态码（整数类型）
- `message`: 错误描述信息（字符串类型）
- ❌ 不使用嵌套的 `error` 对象
- ❌ 不包含 `type` 字段
- ❌ 不包含 `details` 对象
- ⚠️  E2B使用简单的两字段格式，AdNegator必须100%兼容

**示例 - RateLimitError响应**:
```json
{
  "code": 429,
  "message": "Rate limit exceeded: 6 requests in 1 second"
}
```

**HTTP响应头** (RateLimitError):
```
HTTP/1.1 429 Too Many Requests
Retry-After: 60
X-RateLimit-Limit: 5
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1698765432
Content-Type: application/json
```

### 4.2 字段说明 (E2B Official)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `code` | integer | ✅ | HTTP状态码（如 400, 401, 404, 429, 502, 507） |
| `message` | string | ✅ | 错误描述（人类可读，英文） |

**重要说明**:
- ✅ E2B官方仅使用 `code` (integer) 和 `message` (string) 两个字段
- ❌ 不使用嵌套的 `error` 对象
- ❌ 不使用 `type` 字段（SDK根据HTTP状态码自动映射到错误类型）
- ❌ 不使用 `details` 对象
- 💡 SDK内部会根据 `code` 值创建对应的错误类型实例

## 5. SDK错误处理示例

### 5.1 TypeScript SDK错误处理

**按错误类型处理**:
```typescript
import {
  Sandbox,
  RateLimitError,
  AuthenticationError,
  TimeoutError,
  CommandExitError
} from '@adnegator/sdk'

try {
  const sandbox = await Sandbox.create('python-3.11')
  const result = await sandbox.commands.run('python script.py')

} catch (error) {
  // 按SDK错误类型处理
  if (error instanceof RateLimitError) {
    console.error(`Rate limit exceeded. Retry after ${error.retryAfter}s`)
    await sleep(error.retryAfter * 1000)
    // 重试逻辑

  } else if (error instanceof AuthenticationError) {
    console.error('Authentication failed. Check your API key.')
    // 引导用户重新配置凭证

  } else if (error instanceof TimeoutError) {
    console.error('Request timed out. Retrying...')
    // 重试或增加超时时间

  } else if (error instanceof CommandExitError) {
    console.error(`Command failed with exit code ${error.exitCode}`)
    console.error(`Stdout: ${error.stdout}`)
    console.error(`Stderr: ${error.stderr}`)
    // 处理命令失败

  } else {
    console.error(`Unknown error: ${error.message}`)
  }
}
```

**按错误名称处理**:
```typescript
try {
  const sandbox = await Sandbox.create('base', {
```

### 5.3 服务端错误映射

**FastAPI错误处理器**:
```python
from fastapi import FastAPI, HTTPException, Request
from fastapi.responses import JSONResponse
from typing import Any, Dict

app = FastAPI()

# SDK错误类型到HTTP状态码映射
ERROR_STATUS_MAP = {
    'TimeoutError': 502,
    'InvalidArgumentError': 400,
    'NotEnoughSpaceError': 507,
    'NotFoundError': 404,
    'AuthenticationError': 401,
    'TemplateError': 400,
    'RateLimitError': 429,
    'BuildError': 500,
    'FileUploadError': 500,
    'CommandExitError': 200,  # 命令执行成功但退出码非零
}

class SandboxError(Exception):
    """基础错误类"""
    def __init__(
        self,
        error_type: str,
        code: str,
        message: str,
        details: Dict[str, Any] = None,
        **kwargs
    ):
        self.error_type = error_type
        self.code = code
        self.message = message
        self.details = details or {}
        self.extra = kwargs
        super().__init__(message)

@app.exception_handler(SandboxError)
async def sandbox_error_handler(request: Request, exc: SandboxError):
    """统一错误响应格式"""
    response_data = {
        "error": {
            "type": exc.error_type,
            "code": exc.code,
            "message": exc.message,
            "details": exc.details
        }
    }

    # RateLimitError专属字段
    if exc.error_type == 'RateLimitError' and 'retry_after' in exc.extra:
        response_data["error"]["retry_after"] = exc.extra['retry_after']

    status_code = ERROR_STATUS_MAP.get(exc.error_type, 500)

    # 设置响应头
    headers = {}
    if exc.error_type == 'RateLimitError':
        headers['Retry-After'] = str(exc.extra.get('retry_after', 60))
        if 'rate_limit_info' in exc.extra:
            info = exc.extra['rate_limit_info']
            headers['X-RateLimit-Limit'] = str(info.get('limit', ''))
            headers['X-RateLimit-Remaining'] = str(info.get('remaining', ''))
            headers['X-RateLimit-Reset'] = str(info.get('reset', ''))

    return JSONResponse(
        status_code=status_code,
        content=response_data,
        headers=headers
    )

# 使用示例
@app.post("/sandboxes")
async def create_sandbox(template_id: str):
    # 检查速率限制
    if not check_rate_limit(request.user_id):
        raise SandboxError(
            error_type='RateLimitError',
            code='rate_limit_exceeded',
            message='Rate limit exceeded: 6 requests in 1 second',
            retry_after=60,
            rate_limit_info={
                'limit': 5,
                'remaining': 0,
                'reset': int(time.time()) + 60
            }
        )

    # 检查配额
    if user.usage >= user.quota:
        raise SandboxError(
            error_type='RateLimitError',
            code='quota_exceeded',
            message=f'Monthly sandbox quota exceeded: {user.usage}/{user.quota}',
            details={
                'current_usage': user.usage,
                'quota_limit': user.quota,
                'plan': user.plan
            }
        )

    # 创建sandbox
    try:
        sandbox = create_firecracker_vm(template_id)
        return {"sandbox_id": sandbox.id}
    except OrchestratorError as e:
        raise SandboxError(
            error_type='SandboxError',
            code='orchestrator_error',
            message=f'Failed to create sandbox: {str(e)}',
            details={'orchestrator_error': str(e)}
        )
```

### 5.4 gRPC错误处理

**Connect RPC服务端错误映射**:
```python
from connectrpc import ConnectError, Code

def map_to_grpc_error(error: Exception) -> ConnectError:
    """将内部错误映射为gRPC错误"""

    ERROR_CODE_MAP = {
        'TimeoutError': Code.DEADLINE_EXCEEDED,
        'InvalidArgumentError': Code.INVALID_ARGUMENT,
        'NotFoundError': Code.NOT_FOUND,
        'AuthenticationError': Code.UNAUTHENTICATED,
        'RateLimitError': Code.RESOURCE_EXHAUSTED,
        'NotEnoughSpaceError': Code.RESOURCE_EXHAUSTED,
    }

    if isinstance(error, SandboxError):
        grpc_code = ERROR_CODE_MAP.get(error.error_type, Code.INTERNAL)

        return ConnectError(
            code=grpc_code,
            message=error.message,
            details=[
                {
                    'type': 'google.rpc.ErrorInfo',
                    'reason': error.code.upper(),
                    'domain': 'adnegator.com',
                    'metadata': {
                        'error_type': error.error_type,
                        'error_code': error.code
                    }
                }
            ]
        )

    return ConnectError(
        code=Code.INTERNAL,
        message='Internal server error'
    )

# 在gRPC handler中使用
async def start_process(request, context):
    try:
        process = await sandbox.start_command(request.cmd)
        return StartResponse(pid=process.pid)

    except NotFoundError as e:
        raise map_to_grpc_error(
            SandboxError('NotFoundError', 'sandbox_not_found', str(e))
        )

    except Exception as e:
        raise map_to_grpc_error(e)
```

## 附录

### A. E2B 兼容性

本错误响应格式**100%兼容** E2B Official API。

**E2B官方错误响应格式**:
```json
{
  "code": 404,
  "message": "Sandbox with ID 'sbx_xxx' not found"
}
```

**关键兼容性要求**:
- ✅ 使用简单的两字段格式: `code` (integer) + `message` (string)
- ✅ `code` 字段为HTTP状态码（整数类型）
- ✅ `message` 字段为错误描述（字符串类型）
- ❌ 不使用嵌套的 `error` 对象
- ❌ 不包含 `type` 或 `details` 字段

**SDK错误映射**:
SDK会根据HTTP状态码自动创建对应的错误类型实例:

| HTTP状态码 | SDK错误类型 |
|-----------|-------------|
| 400 | InvalidArgumentError |
| 401 | AuthenticationError |
| 404 | NotFoundError |
| 429 | RateLimitError |
| 502 | TimeoutError |
| 507 | NotEnoughSpaceError |

### B. 错误码快速查询

**按 HTTP 状态码**:
- 400: `invalid_parameters`, `weak_password`
- 401: `unauthorized`
- 403: `insufficient_permissions`, `quota_exceeded`, `path_forbidden`, `api_key_limit_exceeded`
- 404: `sandbox_not_found`, `template_not_found`, `user_not_found`
- 408: `process_timeout`
- 409: `invalid_state`, `template_in_use`, `email_already_exists`
- 413: `file_too_large`
- 429: `rate_limit_exceeded`, `process_limit_exceeded`
- 500: `internal_error`, `database_error`, `orchestrator_error`, `s3_error`
- 507: `disk_full`
