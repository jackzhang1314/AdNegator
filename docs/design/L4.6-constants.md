## 1. 超时常量 (Timeout Constants)

### 1.1 请求超时

**用途**: HTTP/gRPC单次请求的超时时间

```typescript
// TypeScript
const REQUEST_TIMEOUT_MS = 60_000  // 60秒 (60,000毫秒)
```

```python
# Python
REQUEST_TIMEOUT = 60.0  # 60秒 (浮点数)
```

**适用范围**:
- REST API调用 (GET, POST, PUT, DELETE)
- gRPC单次RPC调用 (非流式)
- 文件操作 (read, write, stat, list)
- 进程管理 (list, kill)

**配置方式**:
```typescript
// 全局配置
const sandbox = await Sandbox.create('base', {
  requestTimeoutMs: 60000
})

// 单个操作配置
const result = await sandbox.commands.run('echo hello', {
  requestTimeoutMs: 30000  // 覆盖全局配置
})
```

---

### 1.2 命令超时

**用途**: 命令执行的最大等待时间

```typescript
const DEFAULT_COMMAND_TIMEOUT_MS = 60_000  // 60秒
```

**适用范围**:
- `commands.run()` - 等待命令完成的超时
- `pty.run()` - PTY命令超时

**配置方式**:
```typescript
const result = await sandbox.commands.run('python long_script.py', {
  timeoutMs: 300000  // 5分钟
})
```

**注意**:
- `timeoutMs` 是命令执行超时
- `requestTimeoutMs` 是HTTP/gRPC请求超时
- 两者独立计算

---

### 1.3 Sandbox超时

**用途**: Sandbox自动销毁前的存活时间

```typescript
const DEFAULT_SANDBOX_TIMEOUT_MS = 300_000  // 300秒 (5分钟)
```

**计划限制**:
```typescript
const SANDBOX_TIMEOUT_LIMITS = {
  hobby: 3_600_000,      // 1小时 (Hobby计划)
  pro: 86_400_000,       // 24小时 (Pro计划)
  enterprise: Infinity   // 无限制 (Enterprise计划)
}
```

**配置方式**:
```typescript
const sandbox = await Sandbox.create('base', {
  timeoutMs: 3600000  // 1小时
})

// 运行时更新
await sandbox.setTimeout(7200000)  // 2小时
```

---

### 1.4 进程连接超时

**用途**: 连接到运行中进程的超时时间

```typescript
const DEFAULT_PROCESS_CONNECTION_TIMEOUT = 60_000  // 60秒
```

**适用场景**:
- `commands.connect(pid)` - 重连到后台进程

## 2. Keep-Alive常量

### 2.1 Ping间隔

**用途**: HTTP/2连接的心跳间隔

```typescript
const KEEPALIVE_PING_INTERVAL_SEC = 50  // 50秒
```

**HTTP Header**:
```
Keepalive-Ping-Interval: 50
```

**说明**:
- 防止长时间空闲连接被中断
- 特别适用于流式gRPC连接
- 小于大多数负载均衡器的60秒超时

---

### 2.2 连接池配置 (Python)

```python
# HTTP连接池大小
MAX_KEEPALIVE_CONNECTIONS = 40
MAX_CONNECTIONS = 40

# Keep-Alive过期时间
KEEPALIVE_EXPIRY_SEC = 300  # 5分钟
```

**使用 (Python httpx)**:
```python
import httpx

limits = httpx.Limits(
    max_keepalive_connections=40,
    max_connections=40,
    keepalive_expiry=300.0
)

client = httpx.AsyncClient(limits=limits)
```

## 3. ENVD版本常量

```typescript
// TypeScript
const ENVD_VERSION_RECURSIVE_WATCH = '0.1.4'  // 递归目录监视

const ENVD_DEFAULT_USER = '0.4.0'             // 默认用户支持

```

```python
# Python
ENVD_VERSION_RECURSIVE_WATCH = "0.1.4"
ENVD_COMMANDS_STDIN = "0.3.0"
ENVD_DEFAULT_USER = "0.4.0"
ENVD_DEBUG_FALLBACK = "99.99.99"
```

### 3.2 版本检查用法

```typescript
import { compareVersions } from '@adnegator/sdk'

const sandboxInfo = await sandbox.getInfo()
const envdVersion = sandboxInfo.envdVersion

// 检查递归监视支持
if (compareVersions(envdVersion, ENVD_VERSION_RECURSIVE_WATCH) >= 0) {
  // 支持递归监视
  const watcher = sandbox.files.watch('/project', {
    recursive: true
  })
} else {
  throw new TemplateError(
    `Recursive watch requires envd >= ${ENVD_VERSION_RECURSIVE_WATCH}, ` +
    `got ${envdVersion}`
  )
}
```

## 24. Sandbox状态常量
