- **虚线箭头 (⇢)**: 异步调用

## 2. SEQ-001: 沙盒创建流程

### 2.1 流程描述

**需求来源**: L1-F1.1 创建沙盒
**架构依据**: L2-5.1.1 API Server, L2-5.1.2 Scheduler

**业务场景**：
用户通过 SDK 调用 `Sandbox.create({ template: 'python-3.11' })` 创建新沙盒。

**性能要求**：
- 总耗时 < 2s (P99)
- 沙盒创建成功率 > 99.5%

### 2.2 正常流程时序图

```mermaid
sequenceDiagram
    participant Client as Client SDK
    participant API as API Server
    participant DB as PostgreSQL
    participant Redis as Redis
    participant Celery as Celery Worker
    participant K8s as Kubernetes API
    participant Pod as Sandbox Pod
    participant Envd as envd Daemon

    Note over Client,Envd: SEQ-001: 沙盒创建流程 (预期 < 2s)

    Client->>+API: POST /sandboxes<br/>{templateID, timeout, metadata}
    Note right of API: [10ms] 参数验证

    API->>API: 验证 API Key
    API->>API: 生成 sandbox_id, envd_token

    par 并行操作

        DB-->>-API: OK [50ms]

        Redis-->>-API: OK [5ms]
    end

    API->>+Celery: 发送异步任务<br/>create_sandbox_task(id, template)
    Celery-->>-API: Task ID

    API-->>-Client: 201 Created<br/>{sandboxID, envdAccessToken, domain}
    Note right of Client: [100ms] 立即返回给用户

    Note over Celery,Envd: 后台异步创建沙盒

    Celery->>+K8s: CREATE Pod<br/>image={template}<br/>runtimeClassName=gvisor<br/>env={ENVD_TOKEN}
    K8s-->>-Celery: Pod Created [200ms]

    K8s->>+Pod: Schedule & Start [800ms]

    Pod->>Pod: gVisor runsc 启动容器 [500ms]
    Pod->>+Envd: 启动 envd 进程 [100ms]
    Envd->>Envd: 监听端口 49983 [50ms]
    Envd->>Envd: 初始化 RPC 服务 [50ms]
    Envd-->>-Pod: Ready

    Pod-->>-K8s: Pod Status = Running

    loop 轮询 Pod 状态 (最多 10 次)
        Celery->>+K8s: GET Pod Status
        K8s-->>-Celery: Running [50ms]
    end

    Celery->>+Envd: Health Check (gRPC)
    Envd-->>-Celery: OK [50ms]

    Celery->>+DB: UPDATE sandboxes<br/>SET status='running'
    DB-->>-Celery: OK [50ms]

    Celery->>Redis: DEL lock:sandbox:{id}

    Note over Celery: 任务完成 [总耗时 ~1.5s]
```

### 2.3 流程详解

#### 2.3.1 API Server 处理 (0-100ms)

**步骤 1-3: 请求验证和预处理**
```python
# 伪代码
async def create_sandbox(req: CreateSandboxRequest):
    # 1. 验证 API Key (10ms)
    user = await verify_api_key(req.headers['X-API-Key'])

    # 2. 生成唯一标识 (1ms)
    sandbox_id = f"sbx-{uuid.uuid4().hex[:16]}"
    envd_token = generate_jwt(sandbox_id, expires_in=3600)

    # 3. 并行写入数据库和 Redis (50ms)
    await asyncio.gather(
        db.insert_sandbox(sandbox_id, status='creating'),
        redis.set(f"lock:sandbox:{sandbox_id}", 1, ex=300)
    )

    # 4. 发送异步任务 (5ms)
    task_id = await celery.send_task(
        'create_sandbox_task',
        args=[sandbox_id, req.template_id]
    )

    # 5. 立即返回响应 (5ms)
    return SandboxResponse(
        sandboxID=sandbox_id,
        envdAccessToken=envd_token,
        domain=f"{sandbox_id}.sandboxes.example.com"
    )
```

**关键决策**：
- ✅ **异步返回**：立即返回 `201 Created`，无需等待沙盒就绪
- ✅ **分布式锁**：防止重复创建（幂等性）
- ✅ **JWT Token**：envd 访问令牌，包含 sandbox_id 和过期时间

---

#### 2.3.2 Celery Worker 处理 (100-1500ms)

**步骤 4-10: 异步创建沙盒**
```python
@celery.task
async def create_sandbox_task(sandbox_id: str, template_id: str):
    try:
        # 1. 查询模板信息 (50ms)
        template = await db.get_template(template_id)

        # 2. 创建 K8s Pod (200ms)
        pod_spec = {
            'apiVersion': 'v1',
            'kind': 'Pod',
            'metadata': {
                'name': sandbox_id,
                'namespace': 'gvisor-e2b-sandboxes',
                'labels': {'app': 'sandbox', 'sandbox-id': sandbox_id}
            },
            'spec': {
                'runtimeClassName': 'gvisor',
                'containers': [{
                    'name': 'sandbox',
                    'image': template.image,
                    'env': [{'name': 'ENVD_TOKEN', 'value': envd_token}],
                    'resources': {
                        'requests': {'cpu': '1', 'memory': '2Gi'},
                        'limits': {'cpu': '2', 'memory': '4Gi'}
                    }
                }]
            }
        }
        await k8s.create_namespaced_pod('gvisor-e2b-sandboxes', pod_spec)

        # 3. 轮询 Pod 状态 (最多 10s)
        for _ in range(20):
            pod = await k8s.read_namespaced_pod(sandbox_id, 'gvisor-e2b-sandboxes')
            if pod.status.phase == 'Running':
                break
            await asyncio.sleep(0.5)

        # 4. 健康检查 envd (100ms)
        await envd_health_check(sandbox_id)

        # 5. 更新数据库状态 (50ms)
        await db.update_sandbox_status(sandbox_id, 'running')

    except Exception as e:
        await db.update_sandbox_status(sandbox_id, 'failed')
        raise
```

## 3. SEQ-002: 代码执行流程

### 3.1 流程描述

**需求来源**: L1-F2.1 进程管理
**架构依据**: L2-5.2.1 envd Daemon

**业务场景**：
用户在已创建的沙盒内执行代码，获取实时输出。

### 3.2 时序图

```mermaid
sequenceDiagram
    participant Client as Client SDK
    participant Envd as envd Daemon
    participant Shell as Shell Process

    Note over Client,Shell: SEQ-002: 代码执行流程

    Client->>+Envd: Start(cmd='python script.py')<br/>gRPC Call<br/>Authorization: Bearer {token}

    Envd->>Envd: 验证 JWT Token [5ms]
    Envd->>Envd: 设置 cgroup 限制 [10ms]

    Envd->>+Shell: exec.Command('python', 'script.py')
    Note right of Envd: 创建子进程 [20ms]

    Shell->>Shell: 执行 Python 脚本

    Envd-->>Client: StartResponse<br/>{processID, status='running'}
    Note right of Client: [50ms] 返回进程 ID

    Client->>+Envd: Stream(processID)<br/>gRPC Stream

    loop 实时输出流
        Shell->>Shell: print("output line")
        Shell-->>Envd: stdout: "output line"
        Envd-->>Client: StreamResponse<br/>{type='stdout', data='output line'}
    end

    Shell->>Shell: 脚本执行完成
    Shell-->>-Envd: Exit Code = 0

    Envd-->>-Client: StreamResponse<br/>{type='exit', exitCode=0}

    Envd->>Envd: 清理进程资源

    Envd-->>-Client: Stream Closed
```

### 3.3 流程详解

#### 3.3.1 envd 进程管理

```go
// envd 伪代码
func (s *CommandsServer) Start(ctx context.Context, req *pb.StartRequest) (*pb.StartResponse, error) {
    // 1. 验证 JWT Token
    claims, err := verifyToken(ctx)
    if err != nil {
        return nil, status.Error(codes.Unauthenticated, "invalid token")
    }

    // 2. 生成进程 ID
    processID := uuid.New().String()

    // 3. 设置 cgroup 限制
    cgroup := &cgroups.Config{
        CPU:    req.Resources.CpuLimit,
        Memory: req.Resources.MemoryLimit,
    }

    // 4. 创建进程
    cmd := exec.CommandContext(ctx, req.Cmd, req.Args...)
    cmd.Dir = req.WorkingDir
    cmd.Env = req.Env

    stdout, _ := cmd.StdoutPipe()
    stderr, _ := cmd.StderrPipe()

    // 5. 启动进程
    if err := cmd.Start(); err != nil {
        return nil, err
    }

    // 6. 保存进程信息
    s.processes[processID] = &Process{
        ID:     processID,
        Cmd:    cmd,
        Stdout: stdout,
        Stderr: stderr,
    }

    return &pb.StartResponse{
        ProcessID: processID,
        Status:    "running",
    }, nil
}
```

#### 3.3.2 实时输出流

```go
func (s *CommandsServer) Stream(req *pb.StreamRequest, stream pb.Commands_StreamServer) error {
    proc := s.processes[req.ProcessID]

    // 并行读取 stdout 和 stderr
    go func() {
        scanner := bufio.NewScanner(proc.Stdout)
        for scanner.Scan() {
            stream.Send(&pb.StreamResponse{
                Type: "stdout",
                Data: scanner.Bytes(),
            })
        }
    }()

    go func() {
        scanner := bufio.NewScanner(proc.Stderr)
        for scanner.Scan() {
            stream.Send(&pb.StreamResponse{
                Type: "stderr",
                Data: scanner.Bytes(),
            })
        }
    }()

    // 等待进程结束
    err := proc.Cmd.Wait()
    exitCode := proc.Cmd.ProcessState.ExitCode()

    stream.Send(&pb.StreamResponse{
        Type:     "exit",
        ExitCode: int32(exitCode),
    })

    return nil
}
```

### 4.2 时序图

```mermaid

    S3-->>-Celery: OK [300ms]

    Celery->>+K8s: DELETE Pod
    K8s-->>-Celery: OK [100ms]

    Celery->>+DB: UPDATE status='paused'<br/>checkpoint_url='s3://...'
    DB-->>-Celery: OK

    Note over Celery: 暂停完成 [总耗时 ~2.5s]
```

### 4.3 CRIU 检查点详解

#### 4.3.1 CRIU 命令

```bash
# 在沙盒 Pod 内执行
criu checkpoint \
  --tree $(pgrep -f "python script.py") \
  --images-dir /tmp/checkpoint \
  --shell-job \
  --leave-stopped \
  --tcp-established
```

**参数说明**：
- `--tree`: 检查点整个进程树
- `--images-dir`: 输出目录
- `--shell-job`: 支持 shell 任务
- `--leave-stopped`: 保持进程停止状态
- `--tcp-established`: 保存 TCP 连接

#### 4.3.2 检查点文件结构

```
checkpoint/
├── core-1.img         # 进程内存镜像
├── core-2.img
├── pages-1.img        # 内存页
├── pages-2.img
├── mm-1.img           # 内存映射
├── fdinfo-1.img       # 文件描述符
├── fs-1.img           # 文件系统状态
├── inventory.img      # 元数据
└── stats-dump         # 统计信息
```

**文件大小估算**：
- 1GB 内存进程 → 约 1.2GB 检查点文件（含元数据）
- 压缩后 → 约 400-600MB

## 7. SEQ-006: 文件传输流程

### 7.1 文件上传时序图

```mermaid

    Note over Client,FS: SEQ-006.1: 文件上传流程

        Envd->>FS: Append chunk
    end

    Client->>Envd: UploadRequest<br/>{last_chunk=true}

    Envd->>FS: fsync() (确保持久化)
    Envd->>FS: chmod 0644 (设置权限)

    Envd-->>-Client: UploadResponse<br/>{path, size, checksum}
```

```mermaid
    Envd->>+FS: Open file
    FS-->>-Envd: File handle

    loop 分片传输
        Envd->>FS: Read 64KB chunk
        Envd-->>Client: DownloadResponse<br/>{chunk=bytes}
    end

    Envd->>FS: Close file
    Envd-->>-Client: Stream closed
```

### 8.1 控制平面认证

```mermaid
sequenceDiagram
```

## 9. SEQ-008: 超时清理流程

### 9.1 时序图

```mermaid
sequenceDiagram
    participant Cron as Celery Beat (定时任务)
    participant DB as PostgreSQL
    participant K8s as Kubernetes API
    participant S3 as S3/MinIO

    Note over Cron,S3: SEQ-008: 超时清理流程 (每分钟)

    loop 每 60 秒
        Cron->>Cron: 触发清理任务

        Cron->>+DB: SELECT id, status<br/>FROM sandboxes<br/>WHERE created_at < NOW() - timeout<br/>AND status IN ('running', 'paused')
        DB-->>-Cron: [{id: 'sbx-abc', status: 'running'}, ...]

        loop 每个超时沙盒
            alt 状态为 running
                Cron->>+K8s: DELETE Pod<br/>namespace/name
                K8s-->>-Cron: OK
            else 状态为 paused
                Cron->>+S3: DELETE /checkpoints/{id}/
                S3-->>-Cron: OK
            end

            Cron->>+DB: DELETE FROM sandboxes<br/>WHERE id={id}
            DB-->>-Cron: OK

            Note right of Cron: 记录审计日志
        end
    end
```

### 9.2 清理策略

| 沙盒状态 | 超时时间 | 清理操作 |
|----------|----------|----------|
| **running** | 用户设置 (默认 1h) | 删除 Pod |
| **paused** | 7 天 | 删除 checkpoint 文件 |
| **creating** | 5 分钟 | 删除记录 (创建失败) |
| **terminating** | 10 分钟 | 强制删除 |
