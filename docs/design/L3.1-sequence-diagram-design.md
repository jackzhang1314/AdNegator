## 2. SEQ-001: 沙盒创建流程

### 2.1 流程描述

**需求来源**: L1-F1.1 创建沙盒
**架构依据**: L2-5.1.1 API Server (Go/Gin), L5-2.2 Orchestrator (Go gRPC)
**源码参考**: `/tmp/infra/packages/api/internal/handlers/sandbox.go`, `/tmp/infra/packages/orchestrator/internal/sandbox/sandbox.go`

**业务场景**：
用户通过 SDK 调用 `Sandbox.create({ template: 'python-3.11' })` 创建新沙盒。

**性能要求**：
- 总耗时 < 2s (P99)
- 沙盒创建成功率 > 99.5%

**⚠️ E2B 官方架构特性**：
- ✅ **同步调用**：无 Celery，API 直接调用 Orchestrator gRPC
- ✅ **Firecracker VM**：使用 Firecracker microVM（非 K8s Pod）
- ✅ **Nomad 部署**：Orchestrator 运行在 Nomad System Job（非 K8s）

### 2.3 流程详解

#### 2.3.1 API Server 处理 (0-100ms)

**步骤 1-3: 请求验证和预处理**（基于 E2B 官方 Go 代码）

```go
// internal/handlers/sandbox.go (基于 E2B 官方实现)
func (s *APIStore) PostSandboxes(c *gin.Context, params api.PostSandboxesParams) {
    ctx := c.Request.Context()

    // 1. 认证 - 从中间件获取 team (10ms)
    team := c.Value(auth.TeamContextKey).(*db.Team)

    // 2. 解析请求
    var req api.PostSandboxesJSONRequestBody
    if err := c.BindJSON(&req); err != nil {
        c.JSON(400, gin.H{"error": "invalid request"})
        return
    }

    // 3. 生成唯一标识 (1ms)
    sandboxID := generateSandboxID() // sbx-{16chars}
    envdToken := generateJWT(sandboxID, team.ID, 3600) // JWT with RS256

    // 4. 检查配额 (BR-021: 基于 team.tier 限制)
    if err := s.checkQuota(ctx, team.ID); err != nil {
        c.JSON(429, gin.H{"error": "quota exceeded"})
        return
    }

    // 5. 并行写入数据库和 Redis (50ms)
    var wg sync.WaitGroup
    wg.Add(2)

    go func() {
        defer wg.Done()
        s.db.InsertSandbox(ctx, db.InsertSandboxParams{
            SandboxID:  sandboxID,
            TeamID:     team.ID,
            TemplateID: req.TemplateId,
            Status:     "creating",
        })
    }()

    go func() {
        defer wg.Done()
        s.redis.Set(ctx, fmt.Sprintf("lock:sandbox:%s", sandboxID), 1, 5*time.Minute)
    }()

    wg.Wait()

    // 6. 同步调用 Orchestrator gRPC (无 Celery!)
    orchResp, err := s.orchestrator.CreateSandbox(ctx, &pb.CreateSandboxRequest{
        SandboxId:         sandboxID,
        TeamId:            team.ID,
        TemplateId:        req.TemplateId,
        Vcpu:              team.Tier.Vcpu,              // 从 tier 获取配额
        RamMb:             team.Tier.RamMb,
        TimeoutSeconds:    req.TimeoutSeconds,
        KernelVersion:     req.KernelVersion,
        FirecrackerVersion: req.FirecrackerVersion,
    })
    if err != nil {
        // 失败时更新状态
        s.db.UpdateSandboxStatus(ctx, db.UpdateSandboxStatusParams{
            SandboxID: sandboxID,
            Status:    "failed",
        })
        c.JSON(500, gin.H{"error": "failed to create sandbox"})
        return
    }

    // 7. 更新数据库状态
    s.db.UpdateSandboxStatus(ctx, db.UpdateSandboxStatusParams{
        SandboxID: sandboxID,
        Status:    "running",
        ClientID:  sql.NullString{String: orchResp.ClientId, Valid: true},
    })

    // 8. 写入 ClickHouse 事件 (异步)
    go s.clickhouse.InsertEvent(ctx, clickhouse.SandboxEvent{
        Timestamp:         time.Now(),
        SandboxID:         sandboxID,
        SandboxTeamID:     team.ID,
        SandboxTemplateID: req.TemplateId,
        EventCategory:     "lifecycle",
        EventLabel:        "sandbox_created",
    })

    // 9. 清理分布式锁
    s.redis.Del(ctx, fmt.Sprintf("lock:sandbox:%s", sandboxID))

    // 10. 返回响应
    c.JSON(201, api.Sandbox{
        SandboxId:       sandboxID,
        ClientId:        orchResp.ClientId,
        EnvdAccessToken: envdToken,
        Status:          "running",
    })
}
```

**关键决策**：
- ✅ **同步调用**：直接调用 Orchestrator gRPC，无需 Celery 异步队列
- ✅ **分布式锁**：防止重复创建（幂等性）
- ✅ **JWT Token**：envd 访问令牌，包含 sandbox_id、team_id 和过期时间
- ✅ **配额检查**：基于 `teams.tier` 表限制资源

#### 2.3.2 Orchestrator 处理 (100-1200ms)

**步骤 4-10: 同步创建 Firecracker VM**（基于 E2B 官方实现）

```go
// internal/sandbox/sandbox.go (基于 E2B 官方实现)
func (m *Manager) CreateSandbox(ctx context.Context, cfg *Config) (*Sandbox, error) {
    // 1. 生成客户端 ID (用于连接 envd)
    clientID := generateClientID()

    // 2. 准备文件系统 (Overlay FS + 模板)
    files, err := m.templateStore.PrepareOverlay(cfg.TemplateID, cfg.SandboxID)
    if err != nil {
        return nil, fmt.Errorf("failed to prepare overlay: %w", err)
    }

    // 3. 分配网络资源 (Network Slot Pool)
    networkSlot, err := m.networkPool.Allocate()
    if err != nil {
        return nil, fmt.Errorf("failed to allocate network slot: %w", err)
    }
    // networkSlot: {IP: "10.0.0.5", MAC: "02:FC:00:00:00:05", TAP: "tap5"}

    // 4. 分配 NBD 设备 (Network Block Device Pool)
    nbdDevice, err := m.nbdPool.Allocate()
    if err != nil {
        return nil, fmt.Errorf("failed to allocate NBD device: %w", err)
    }

    // 5. 构建 Firecracker 配置
    fcConfig := &firecracker.Config{
        SocketPath:      fmt.Sprintf("/tmp/fc-%s.sock", cfg.SandboxID),
        KernelImagePath: "/var/lib/firecracker/vmlinux-5.10",
        KernelArgs:      "console=ttyS0 reboot=k panic=1 pci=off",
        Drives: []firecracker.Drive{{
            DriveID:      "rootfs",
            PathOnHost:   files.RootfsPath,     // Overlay rootfs
            IsRootDevice: true,
            IsReadOnly:   false,
        }},
        NetworkInterfaces: []firecracker.NetworkInterface{{
            StaticConfiguration: &firecracker.StaticNetworkConfiguration{
                MacAddress:  networkSlot.MAC,
                HostDevName: networkSlot.TAP,
                IPConfiguration: &firecracker.IPConfiguration{
                    IPAddr: firecracker.IPAddr{IP: net.ParseIP(networkSlot.IP)},
                },
            },
        }},
        MachineCfg: firecracker.MachineCfg{
            VcpuCount:  cfg.Vcpu,
            MemSizeMib: cfg.RamMB,
            HtEnabled:  false,
        },
    }

    // 6. 启动 Firecracker 进程
    vmmCtx, vmmCancel := context.WithCancel(ctx)
    machine, err := firecracker.NewMachine(vmmCtx, fcConfig)
    if err != nil {
        vmmCancel()
        return nil, fmt.Errorf("failed to create machine: %w", err)
    }

    if err := machine.Start(vmmCtx); err != nil {
        vmmCancel()
        return nil, fmt.Errorf("failed to start machine: %w", err)
    }

    // 7. 等待 envd 就绪 (健康检查)
    envdURL := fmt.Sprintf("http://%s:49983", networkSlot.IP)
    if err := waitForEnvd(ctx, envdURL, 60*time.Second); err != nil {
        machine.StopVMM()
        vmmCancel()
        return nil, fmt.Errorf("envd not ready: %w", err)
    }

    // 8. 创建 Sandbox 对象
    sandbox := &Sandbox{
        ID:         cfg.SandboxID,
        ClientID:   clientID,
        TeamID:     cfg.TeamID,
        TemplateID: cfg.TemplateID,
        Process:    machine,
        VMMCtx:     vmmCtx,
        VMMCancel:  vmmCancel,
        NetworkSlot: networkSlot,
        NBDDevice:  nbdDevice,
        Status:     "running",
        CreatedAt:  time.Now(),
    }

    // 9. 注册沙盒到管理器
    m.mu.Lock()
    m.sandboxes[cfg.SandboxID] = sandbox
    m.mu.Unlock()

    // 10. 启动超时清理 goroutine (BR-070)
    if cfg.TimeoutSeconds > 0 {
        go m.scheduleTimeout(sandbox, time.Duration(cfg.TimeoutSeconds)*time.Second)
    }

    return sandbox, nil
}

// waitForEnvd 等待 envd HTTP 健康检查
func waitForEnvd(ctx context.Context, envdURL string, timeout time.Duration) error {
    ctx, cancel := context.WithTimeout(ctx, timeout)
    defer cancel()

    ticker := time.NewTicker(500 * time.Millisecond)
    defer ticker.Stop()

    for {
        select {
        case <-ctx.Done():
            return fmt.Errorf("timeout waiting for envd")
        case <-ticker.C:
            resp, err := http.Get(envdURL + "/")
            if err != nil {
                continue
            }
            resp.Body.Close()
            if resp.StatusCode == 200 {
                return nil
            }
        }
    }
}
```

### 2.5 性能关键路径（E2B 官方架构）

| 步骤 | 耗时 | 累计 | 优化方向 |
|------|------|------|----------|
| API 参数验证 | 10ms | 10ms | - |
| 数据库写入 | 50ms | 60ms | 索引优化 |
| Orchestrator gRPC 调用 | 5ms | 65ms | - |
| 分配网络槽位 | 10ms | 75ms | - |
| 分配 NBD 设备 | 10ms | 85ms | - |
| 准备 Overlay FS | 50ms | 135ms | **模板预热** ✅ |
| 启动 Firecracker | 200ms | 335ms | UFFD 内存后端 |
| 加载 vmlinux 内核 | 100ms | 435ms | - |
| 挂载 rootfs (NBD) | 50ms | 485ms | - |
| 配置网络 (TAP) | 50ms | 535ms | - |
| 启动 envd (PID 1) | 100ms | 635ms | 优化启动脚本 |
| envd 初始化 gRPC | 50ms | 685ms | - |
| 健康检查 | 50ms | 735ms | - |
| 数据库更新 | 50ms | 785ms | - |
| **总耗时** | **~800ms** | ✅ **优于 K8s 方案** |

**性能优势对比**：

| 架构 | 总耗时 | 瓶颈 |
|------|--------|------|
| **E2B (Firecracker)** | **~800ms** | Firecracker 启动 (200ms) |
| 原方案 (K8s + gVisor) | ~1210ms | gVisor runsc 启动 (500ms) |
| **性能提升** | **33%** | ✅ Firecracker 更轻量 |

**优化建议**：
- ✅ **UFFD 内存后端**：使用 User-space Page Fault Handling 减少内存初始化时间
- ✅ **模板预热**：预先准备常用模板的 Overlay FS
- ✅ **网络槽位池**：预分配网络资源池

**业务场景**：
用户在已创建的沙盒内执行代码，获取实时输出。

**⚠️ E2B 官方架构特性**：
- ✅ **Connect RPC over HTTP/2**：使用 Connect RPC（非传统 gRPC）

### 3.3 流程详解

#### 3.3.1 envd 进程管理（基于 E2B 官方 Go 实现）

```go
// internal/services/process/process.go (基于 E2B 官方实现)
package process

import (
    "context"
    "fmt"
    "os/exec"
    "sync"

    "connectrpc.com/connect"
    pb "github.com/e2b-dev/infra/packages/envd/internal/services/spec/process"
)

type Service struct {
    processes map[string]*Process
    mu        sync.RWMutex
    defaults  *execcontext.Defaults
}

type Process struct {
    ID       string
    Tag      string
    Cmd      *exec.Cmd
    Status   string
    ExitCode int
    StartedAt time.Time
}

// Start 启动进程 (对应 POST /process/start)
func (s *Service) Start(
    ctx context.Context,
    req *connect.Request[pb.StartRequest],
) (*connect.Response[pb.StartResponse], error) {
    // 1. BR-040: 检查并发进程数限制
    s.mu.RLock()
    count := len(s.processes)
    s.mu.RUnlock()

    if count >= MaxConcurrentProcesses {
        return nil, connect.NewError(
            connect.CodeResourceExhausted,
            fmt.Errorf("BR-040: max concurrent processes exceeded (%d)", count),
        )
    }

    // 2. 生成进程 ID
    processID := generateProcessID()

    // 3. 构建命令（使用 bash -c 包装）
    cmd := exec.CommandContext(ctx, "/bin/bash", "-c", req.Msg.Process.Cmd)

    // 4. 设置工作目录
    if req.Msg.Process.Cwd != nil {
        cmd.Dir = *req.Msg.Process.Cwd
    } else {
        cmd.Dir = "/home/user"  // E2B 默认工作目录
    }

    // 5. 合并环境变量 (defaults + user)
    envVars := make(map[string]string)
    for k, v := range s.defaults.EnvVars {
        envVars[k] = v
    }
    for k, v := range req.Msg.Process.Envs {
        envVars[k] = v
    }
    cmd.Env = convertEnvMap(envVars)

    // 6. 创建输出管道
    stdoutPipe, _ := cmd.StdoutPipe()
    stderrPipe, _ := cmd.StderrPipe()

    // 7. 启动进程
    if err := cmd.Start(); err != nil {
        return nil, connect.NewError(connect.CodeInternal, err)
    }

    // 8. 保存进程信息
    process := &Process{
        ID:        processID,
        Tag:       getTag(req.Msg.Tag),
        Cmd:       cmd,
        Status:    "running",
        StartedAt: time.Now(),
    }

    s.mu.Lock()
    s.processes[processID] = process
    s.mu.Unlock()

    // 9. 异步等待进程结束
    go func() {
        err := cmd.Wait()
        process.Status = "finished"
        if err != nil {
            process.ExitCode = cmd.ProcessState.ExitCode()
        }
    }()

    return connect.NewResponse(&pb.StartResponse{
        ProcessId: processID,
        Status:    "running",
    }), nil
}
```

#### 3.3.2 实时输出流（基于 E2B 官方实现）

```go
// SubscribeOutput 订阅进程输出流 (对应 POST /process/subscribe)
func (s *Service) SubscribeOutput(
    ctx context.Context,
    req *connect.Request[pb.SubscribeOutputRequest],
    stream *connect.ServerStream[pb.SubscribeOutputResponse],
) error {
    s.mu.RLock()
    process, exists := s.processes[req.Msg.ProcessId]
    s.mu.RUnlock()

    if !exists {
        return connect.NewError(connect.CodeNotFound, fmt.Errorf("process not found"))
    }

    // 实时流式传输 stdout 和 stderr
    errChan := make(chan error, 2)

    // 读取 stdout
    go func() {
        scanner := bufio.NewScanner(process.Cmd.Stdout)
        for scanner.Scan() {
            line := scanner.Text()
            if err := stream.Send(&pb.SubscribeOutputResponse{
                Type: "stdout",
                Line: line,
            }); err != nil {
                errChan <- err
                return
            }
        }
        errChan <- scanner.Err()
    }()

    // 读取 stderr
    go func() {
        scanner := bufio.NewScanner(process.Cmd.Stderr)
        for scanner.Scan() {
            line := scanner.Text()
            if err := stream.Send(&pb.SubscribeOutputResponse{
                Type: "stderr",
                Line: line,
            }); err != nil {
                errChan <- err
                return
            }
        }
        errChan <- scanner.Err()
    }()

    // 等待进程结束
    err := <-errChan

    // 发送退出码
    stream.Send(&pb.SubscribeOutputResponse{
        Type:     "exit",
        ExitCode: int32(process.ExitCode),
    })

    return err
}
```

```go
func (m *Manager) scheduleTimeout(sandbox *Sandbox, timeout time.Duration) {
    // 使用 time.Sleep 等待超时
    timer := time.NewTimer(timeout)
    defer timer.Stop()

    select {
    case <-timer.C:
        // 超时到期，执行清理
        log.Printf("Sandbox %s timed out after %v", sandbox.ID, timeout)

        // 1. 停止 Firecracker VM
        if err := sandbox.Process.StopVMM(); err != nil {
            log.Printf("Failed to stop VMM: %v", err)
        }
        sandbox.VMMCancel()

        // 2. 释放资源
        m.networkPool.Release(sandbox.NetworkSlot)
        m.nbdPool.Release(sandbox.NBDDevice)
        sandbox.Files.Cleanup()

        // 3. 从 manager 删除
        m.mu.Lock()
        delete(m.sandboxes, sandbox.ID)
        m.mu.Unlock()

        // 4. 通知 API Server (可选)
        // 可以通过 webhook 或 gRPC 回调通知 API
        // m.notifyAPI(sandbox.ID, "timeout")

    case <-sandbox.VMMCtx.Done():
        // 沙盒被手动删除，取消超时
        log.Printf("Sandbox %s deleted before timeout", sandbox.ID)
        return
    }
}
```

| **creating** | 5 分钟 | 健康检查失败 | 在 SEQ-001 中处理 |
| **terminating** | 无 | 同步删除 | 在 SEQ-005 中处理 |

**关键差异**：
- ❌ **无全局扫描**：不需要定期扫描数据库
- ✅ **独立计时**：每个沙盒有独立的超时 goroutine
- ✅ **低开销**：goroutine 非常轻量（~2KB 内存）
- ✅ **取消机制**：手动删除时自动取消超时

**性能优势**：
- ✅ **无轮询开销**：避免每分钟扫描数据库
- ✅ **精确超时**：精确到秒级
- ✅ **可扩展性**：支持数十万并发沙盒（goroutine 非常轻量）

## 10. 异常处理流程

### 10.1 数据库连接失败

```mermaid
sequenceDiagram
    participant API as API Server
    participant DB as PostgreSQL
    participant Client as Client SDK

    API->>+DB: INSERT INTO sandboxes
    DB-->>-API: Error: Connection refused

    API->>API: 重试 3 次 (指数退避)

    alt 重试成功
        DB-->>API: OK
        API-->>Client: 201 Created
    else 重试失败
        API->>API: 记录错误日志
        API-->>Client: 500 Internal Server Error<br/>{error: 'database_unavailable'}
    end
```

### 10.2 Kubernetes API 失败

```mermaid
sequenceDiagram
```
