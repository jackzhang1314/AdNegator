## 1. API 概述

### 1.1 Base URL

```
Production:  https://api.gvisor-e2b.com
Staging:     https://api-staging.gvisor-e2b.com
Development: http://localhost:8000
```

### 1.2 协议

- **Control Plane**: REST API over HTTPS
- **Data Plane**: Connect RPC (gRPC over HTTP/2)

### 1.3 E2B 兼容性声明

本 API **100% 兼容** E2B API 规范，现有 E2B 客户端仅需更改 Base URL 即可无缝迁移。

**兼容版本**: E2B API v1 (2024-11)

## 2. 认证方式

### 2.1 Control Plane 认证 (API Key)

**Header**:
```
X-API-Key: sk_abc123...
```

**获取方式**:
1. 注册账户
2. 在控制台生成 API Key

**示例**:
```bash
curl -X POST https://api.gvisor-e2b.com/sandboxes \
  -H "X-API-Key: sk_abc123..." \
  -H "Content-Type: application/json" \
  -d '{"templateID": "python-3.11"}'
```

### 2.2 Data Plane 认证 (Bearer Token)

**Header**:
```
Authorization: Bearer eyJhbGciOi...
```

**Token 来源**: 创建沙盒时返回的 `envdAccessToken`

**示例**:
```javascript
const client = createConnectTransport({
  baseUrl: `https://${sandboxDomain}:49983`,
  httpVersion: "2",
  headers: {
    Authorization: `Bearer ${envdAccessToken}`
  }
});
```

## 3. 通用规范

### 3.1 请求格式

**Content-Type**: `application/json`

**通用字段**:
```json
{
  "templateID": "string",   // 模板 ID（可选）
  "timeout": 3600,          // 超时秒数（可选）
  "metadata": {},           // 用户元数据（可选）
  "autoPause": false        // E2B 兼容字段（可选）
}
```

### 3.2 响应格式

**成功响应**:
```json
{
  "sandboxID": "sbx_abc123",
  "status": "running",
  "...": "..."
}
```

**错误响应** (详见 7. 错误响应):
```json
{
  "error": {
    "code": "sandbox_not_found",
    "message": "Sandbox not found",
    "details": {}
  }
}
```

### 3.3 分页

**查询参数**:
```
?page=1&page_size=20
```

**响应格式**:
```json
{
  "data": [...],
  "pagination": {
    "page": 1,
    "page_size": 20,
    "total": 150,
    "total_pages": 8
  }
}
```

### 3.4 时间格式

**ISO 8601 with timezone**:
```
2025-11-05T12:34:56.789Z
```

## 4. 沙盒管理 API

### 4.1 创建沙盒

**Endpoint**: `POST /sandboxes`

**描述**: 创建新的代码执行沙盒

**对应需求**: L1-F1.1 创建沙盒
**对应业务规则**: BR-020, BR-021, BR-022

**请求体**:
```json
{
  "templateID": "python-3.11",      // 可选，默认 "base"
  "timeout": 3600,                   // 可选，默认 3600 秒
  "metadata": {                      // 可选
    "project": "AI Agent",
    "env": "dev"
  },
  "secure": true,                    // E2B 兼容，生成 envdAccessToken
  "autoPause": false                 // E2B 兼容，自动暂停（暂不支持）
}
```

**响应** (201 Created):
```json
{
  "sandboxID": "sbx_a1b2c3d4e5f6g7h8",
  "envdVersion": "0.2.0",
  "envdAccessToken": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9...",
  "domain": "sbx-a1b2c3d4e5f6g7h8.sandboxes.gvisor-e2b.com"
}
```

**cURL 示例**:
```bash
curl -X POST https://api.gvisor-e2b.com/sandboxes \
  -H "X-API-Key: sk_abc123..." \
  -H "Content-Type: application/json" \
  -d '{
    "templateID": "python-3.11",
    "timeout": 3600,
    "metadata": {"project": "test"}
  }'
```

**错误码**:
- `400` - 参数错误 (invalid_parameters)
- `401` - 认证失败 (unauthorized)
- `403` - 配额超限 (quota_exceeded)
- `500` - 服务器错误 (internal_error)

---

### 4.2 获取沙盒信息

**Endpoint**: `GET /sandboxes/{sandboxID}`

**描述**: 查询沙盒详细信息

**路径参数**:
- `sandboxID` (string, required): 沙盒 ID

**响应** (200 OK):
```json
{
  "sandboxID": "sbx_a1b2c3d4e5f6g7h8",
  "status": "running",
  "templateID": "python-3.11",
  "domain": "sbx-a1b2c3d4e5f6g7h8.sandboxes.gvisor-e2b.com",
  "createdAt": "2025-11-05T12:00:00.000Z",
  "updatedAt": "2025-11-05T12:00:05.000Z",
  "metadata": {
    "project": "test"
  }
}
```

**状态说明**:
- `creating`: 创建中
- `running`: 运行中
- `paused`: 已暂停
- `resuming`: 恢复中
- `terminating`: 销毁中
- `failed`: 失败

**错误码**:
- `404` - 沙盒不存在 (sandbox_not_found)

---

### 4.3 列出沙盒

**Endpoint**: `GET /sandboxes`

**描述**: 查询用户的沙盒列表

**查询参数**:
- `status` (string, optional): 状态筛选 (running, paused, etc.)
- `page` (integer, optional): 页码，默认 1
- `page_size` (integer, optional): 每页数量，默认 20

**响应** (200 OK):
```json
{
  "data": [
    {
      "sandboxID": "sbx_xxx",
      "status": "running",
      "templateID": "python-3.11",
      "createdAt": "2025-11-05T12:00:00.000Z"
    }
  ],
  "pagination": {
    "page": 1,
    "page_size": 20,
    "total": 5,
    "total_pages": 1
  }
}
```

---

### 4.4 暂停沙盒

**Endpoint**: `POST /sandboxes/{sandboxID}/pause`

**描述**: 暂停运行中的沙盒（使用 CRIU 检查点）

**对应需求**: L1-F1.3 暂停沙盒
**对应业务规则**: BR-060

**路径参数**:
- `sandboxID` (string, required): 沙盒 ID

**请求体**: 无

**响应** (204 No Content):
```
(空响应体)
```

**错误码**:
- `404` - 沙盒不存在 (sandbox_not_found)
- `409` - 已经暂停 (already_paused)
- `409` - 无法暂停（状态不允许）(invalid_state)

**注意**:
- 只有 `running` 状态的沙盒可以暂停
- 暂停后 CPU 资源释放，但保留检查点文件

---

### 4.5 连接沙盒（恢复）

**Endpoint**: `POST /sandboxes/{sandboxID}/connect`

**描述**: 连接到沙盒，如果已暂停则自动恢复

**对应需求**: L1-F1.2 连接沙盒
**对应业务规则**: BR-060

**路径参数**:
- `sandboxID` (string, required): 沙盒 ID

**请求体**: 无

**响应**:
- **200 OK** (已在运行): 无需恢复
  ```json
  {
    "sandboxID": "sbx_xxx",
    "status": "running",
    "domain": "sbx-xxx.sandboxes.gvisor-e2b.com"
  }
  ```

- **201 Created** (从暂停恢复): 已触发恢复
  ```json
  {
    "sandboxID": "sbx_xxx",
    "status": "resuming",
    "domain": "sbx-xxx.sandboxes.gvisor-e2b.com"
  }
  ```

**错误码**:
- `404` - 沙盒不存在 (sandbox_not_found)
- `500` - 恢复失败 (restore_failed)

---

### 4.6 删除沙盒

**Endpoint**: `DELETE /sandboxes/{sandboxID}`

**描述**: 删除沙盒并清理所有资源

**对应需求**: L1-F1.4 销毁沙盒

**路径参数**:
- `sandboxID` (string, required): 沙盒 ID

**响应** (204 No Content):
```
(空响应体)
```

**清理内容**:
- Kubernetes Pod
- CRIU 检查点文件（如果存在）
- 数据库记录（软删除）

**错误码**:
- `404` - 沙盒不存在 (sandbox_not_found)

## 6. MCP Gateway API

### 6.1 概述

MCP (Model Context Protocol) Gateway API 用于管理沙箱中的 MCP 工具集成。支持 200+ 预验证工具（来自 Docker 官方 MCP 目录）。

**特性**:
- ✅ 创建沙箱时配置 MCP 工具
- ✅ 运行时动态添加/移除工具
- ✅ 获取 MCP Gateway URL（用于 AI 客户端连接）
- ✅ 浏览 MCP 服务器目录

---

### 6.2 创建带 MCP 的沙箱

**端点**: `POST /v1/sandboxes`（扩展）

**请求** (扩展 `mcp` 字段):
```json
{
  "templateID": "base",
  "timeout": 300,
  "metadata": {},
  "mcp": {
    "browserbase": {
      "apiKey": "sk_...",
      "geminiApiKey": "AIza...",
      "projectId": "proj_..."
    },
    "exa": {
      "apiKey": "exa_..."
    },
    "github": {
      "personalAccessToken": "ghp_..."
    }
  }
}
```

**`mcp` 字段说明**:
- **类型**: `object` (可选)
- **键**: MCP 服务器 ID（如 `browserbase`, `exa`, `github`）
- **值**: 该服务器所需的凭证 object（根据服务器的 `credential_schema` 定义）

**响应** (扩展):
```json
{
  "sandboxID": "sbx_abc123",
  "templateID": "base",
  "alias": "fancy-unicorn-42",
  "clientID": "client_xyz789",
  "envdDomain": "sbx-abc123.e2b.dev",
  "envdPort": 49983,
  "envdAccessToken": "eyJhbGciOi...",
  "createdAt": "2025-11-05T10:00:00Z",
  "mcp": {
    "enabled": true,
    "servers": ["browserbase", "exa", "github"]
  }
}
```

**错误码**:
- `400` - 无效的 MCP 配置
  - `invalid_mcp_server`: 服务器 ID 不存在
  - `invalid_credentials`: 凭证不符合 schema
  - `credentials_encryption_failed`: 凭证加密失败

**对应 L1 需求**: F7.2（沙箱 MCP 配置）

---

### 6.3 获取 MCP Gateway URL

**端点**: `GET /v1/sandboxes/{sandboxID}/mcp-url`

**路径参数**:
- `sandboxID` - 沙箱 ID

**响应** (200 OK):
```json
{
  "url": "https://sandbox-sbxabc123.e2b.dev/mcp",
  "token": "Bearer tok_7a8b9c0d1e2f3g4h",
  "expiresAt": "2025-11-05T15:00:00Z",
  "servers": ["browserbase", "exa", "github"]
}
```

**响应字段**:
| 字段 | 类型 | 说明 |
|------|------|------|
| `url` | string | MCP Gateway 公开 URL |
| `token` | string | Bearer token（用于 AI 客户端认证） |
| `expiresAt` | string | Token 过期时间（ISO 8601） |
| `servers` | array | 已配置的 MCP 服务器列表 |

**错误码**:
- `404` - 沙箱不存在 (sandbox_not_found)
- `400` - MCP 未启用 (mcp_not_enabled)

**使用示例** (TypeScript):
```typescript
// 1. 创建沙箱
const sandbox = await fetch('https://api.gvisor-e2b.com/v1/sandboxes', {
  method: 'POST',
  headers: {
    'X-API-Key': 'sk_...',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    templateID: 'base',
    mcp: {
      browserbase: {
        apiKey: 'sk_...',
        geminiApiKey: 'AIza...',
        projectId: 'proj_...'
      }
    }
  })
}).then(r => r.json());

// 2. 获取 MCP URL
const mcpInfo = await fetch(
  `https://api.gvisor-e2b.com/v1/sandboxes/${sandbox.sandboxID}/mcp-url`,
  {
    headers: { 'X-API-Key': 'sk_...' }
  }
).then(r => r.json());

console.log('Connect your AI client to:', mcpInfo.url);
console.log('Use token:', mcpInfo.token);

// 3. AI 客户端（如 Claude Desktop）连接到 mcpInfo.url
```

**对应 L1 需求**: F7.3（MCP Gateway）

### 7.1 协议说明

envd 使用 **Connect RPC** (gRPC over HTTP/2)

**Protocol Buffers 定义** (简化版):
```protobuf
syntax = "proto3";

package commands.v1;

service CommandsService {
  rpc Start(StartRequest) returns (StartResponse);
  rpc Stream(StreamRequest) returns (stream StreamResponse);
  rpc Kill(KillRequest) returns (KillResponse);
}

message StartRequest {
  string cmd = 1;
  repeated string args = 2;
  string working_dir = 3;
  map<string, string> env = 4;
  int32 timeout = 5;
}

message StartResponse {
  string process_id = 1;
  string status = 2;
}

message StreamRequest {
  string process_id = 1;
}

message StreamResponse {
  string type = 1;  // "stdout", "stderr", "exit"

}
```

### A. SDK 使用示例

#### TypeScript SDK

```typescript
import { Sandbox } from '@gvisor-e2b/sdk';

// 配置 Base URL
const apiUrl = 'https://api.gvisor-e2b.com';
const apiKey = process.env.API_KEY;

// 创建沙盒
const sandbox = await Sandbox.create({
  apiUrl,
  apiKey,
  template: 'python-3.11',
  timeoutMs: 3600000  // 1 hour
});

// 执行代码
const proc = await sandbox.commands.start({
  cmd: 'python',
  args: ['script.py']
});

// 实时输出
proc.stdout.on('data', (data) => {
  console.log(data.toString());
});

await proc.wait();

// 清理
await sandbox.kill();
```

```python
from gvisor_e2b import Sandbox

# 创建沙盒
sandbox = Sandbox.create(
    api_url="https://api.gvisor-e2b.com",

proc = sandbox.commands.start(cmd="python", args=["script.py"])

    print(line)

sandbox.kill()
```
